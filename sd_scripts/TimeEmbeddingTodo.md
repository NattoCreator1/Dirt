## 1. 当前脚本已覆盖的部分

### 1.1 静态 Inpainting 合成与强标注闭环
当前脚本执行流程为：随机采样单帧 clean 底图与单帧 mask，经 Inpainting 生成单张合成图像，并从 mask 直接导出 `tile_cov` 与 `s_simple、S_full、L` 等标签[2][3][5]。这对应大纲中 SD 静态合成的核心要求，即局部编辑与标签天然对齐[1][2]。

## 2. 尚未覆盖的时序 embedding 相关要点

### 2.1 未生成“连续帧序列”数据
大纲中 SD_seq 的数据形式为长度为 T 的序列样本，包含每一帧的合成图与强标注标签集合[2][3]；当前脚本按样本独立采样单帧，不存在同一视频片段的连续帧输入与输出[5]。

### 2.2 未实现 mask 的时序演化机制
大纲明确提出用 warp 位移场与扰动项使 mask 随时间平滑演化，并用相邻帧 IoU、质心轨迹、覆盖率曲线等硬指标验证时序合理性[1][2][3]。当前脚本仅使用单帧 mask，不包含演化规则与时序一致性指标[5]。

### 2.3 未产出时序训练所需的元数据
大纲建议为时序建模记录 `Δt`、事件类型、以及 `ΔS_t` 等信息，以支持时序损失与 embedding 设计[3]。当前脚本 manifest 记录了生成参数与质量分数，但未保存序列 id、帧序号、相邻帧对应关系、以及 `ΔS` 等时序监督字段[5]。

### 2.4 未接入时序损失与时序编码模块
大纲中的时序学习包含两种典型实现：基于已知 warp 的一致性约束，或基于 `S_t` 的变化率监督形式的时序损失[1][2][3]。当前脚本仅负责数据生成，不涉及训练侧的序列 DataLoader、时序编码模块、以及 `L_temp` 的损失实现[2][3][5]。

## 3. 将时序 embedding 纳入 SD 增强的最小补齐路径

### 3.1 新增 SD_seq 生成脚本或生成模式
按大纲实现 SD_seq 的最小工程方案为：对同一段 clean 视频连续帧构造序列，生成初始 mask 后用 warp 与扰动得到 `mask_t`，逐帧 Inpainting 并保持背景稳定性可验证[1][2][3]。产物建议按序列落盘，例如 `seq_id/frame_idx/image/mask/tile_cov/S_full`。

### 3.2 新增时序训练接口
训练侧需要 DataLoader 输出 clip，而不是独立帧，并启用大纲给出的时序一致性损失或变化率监督损失，使时序 embedding 有监督信号可学习[1][2][3]。

## 4. 当前阶段的合理定位

当前 Inpainting 生成器可以作为 SD 增强闭环的“静态合成基座”，用于先完成 `Real only` 与 `Real + Syn` 的增益对照，以及后续 `q_D + q_T` 筛选加权闭环[1][2][3][5]。时序 embedding 的实现属于下一阶段工作，需要以 SD_seq 的序列强标注数据为前置条件[1][2][3]。
