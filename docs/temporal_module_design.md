## 一、核心洞察与动机

### 1.1 物理观察

车载镜头脏污检测场景中存在一个关键的物理特性：

```
┌─────────────────────────────────────────────────────────────────────┐
│                                                                     │
│  脏污：相对镜头固定（附着在镜头表面）                                  │
│  场景：随车辆移动而变化（道路、建筑、其他车辆）                           │
│                                                                     │
│  → 通过"固定 vs 变化"的时序对比，可以更准确地识别脏污                      │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

### 1.2 现有方法的局限性

| 方法 | 特点 | 局限性 |
|------|------|--------|
| **单帧检测**（大多数现有工作） | 每帧独立判断 | 易将道路物体误判为脏污 |
| **我们的时序方法** | 利用多帧的时序一致性 | 需要视频输入，计算成本略高 |

### 1.3 创新点

利用脏污相对固定的物理特性，通过时序建模：
1. **提升脏污识别精度**：区分固定脏污和移动物体
2. **增强跨域泛化能力**：时序一致性是跨域稳定的特征
3. **提供可解释性**：模型预测基于物理特性而非数据拟合

## 二、技术方案

### 2.1 整体架构

```
┌─────────────────────────────────────────────────────────────────────┐
│                          时序模型架构                                  │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   [I_{t-T}, ..., I_{t-1}, I_t]  →  Shared Backbone (ResNet18)      │
│                                      ↓                              │
│                         [F_{t-T}, ..., F_{t-1}, F_t]                │
│                                      ↓                              │
│                           ┌─────────────────────┐                   │
│                           │   时序编码模块        │                   │
│                           │  (LSTM/Conv/Attn)   │                   │
│                           └─────────────────────┘                   │
│                                      ↓                              │
│                            时序感知特征 F_temporal                   │
│                                      ↓                              │
│                     ┌────────────────┴────────────────┐              │
│                     ↓                                 ↓              │
│              Tile Head                          Global Head           │
│              Ĝ [B,4,8,8]                        Ŝ [B,1]              │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

### 2.2 时序编码模块选择

| 方案 | 架构 | 参数量 | 优点 | 缺点 | 推荐场景 |
|------|------|--------|------|------|----------|
| **LSTM/GRU** | RNN 结构 | 中 | 适合长时序，成熟稳定 | 参数较多，推理较慢 | 长序列建模 |
| **Temporal Conv** | 1D 卷积 | 小 | 轻量高效，推理快 | 只能捕捉短时序 | 实时应用 |
| **Temporal Attn** | Self-Attention | 中 | 可解释性强，并行计算 | 计算成本较高 | 需要可解释性 |

**推荐方案**: Temporal Conv（轻量）+ 可选 Attention（可解释性）

### 2.3 时序损失设计

#### 2.3.1 严重度变化率损失

```
L_temp = (1/B(T-1)) · Σ_{t=1}^{T-1} |(Ŝ_{t+1} - Ŝ_t) - (S'_{t+1} - S'_t)|
```

**物理含义**: 模型学习"脏污随时间变化的速率"，而非简单的平滑

**优势**:
- 直接监督脏污演化动力学
- 适用于累积/消退/突变等不同事件类型

#### 2.3.2 时序一致性损失（简化版）

```
L_consistency = (1/B(T-1)) · Σ_{t=1}^{T-1} ||Ĝ_t - Ĝ_{t-1}||²
```

**物理含义**: 脏污的 tile 预测在时间上应该平滑

**优势**:
- 无需 SD_seq 的时序标注
- 可应用于真实视频数据

### 2.4 单帧/序列兼容设计

时序编码模块设计为"可选启用"：

```python
class TemporalSoilingModel(nn.Module):
    def __init__(self, enable_temporal=False, seq_len=5):
        self.enable_temporal = enable_temporal
        self.backbone = ResNet18(...)
        self.temporal_module = TemporalConv(...) if enable_temporal else None
        self.tile_head = Conv2d(512, 4, 1)
        self.global_head = Linear(512, 1)
        self.aggregator = SeverityAggregator()

    def forward(self, x):
        # x: [B, T, C, H, W] 或 [B, C, H, W]
        if not self.enable_temporal:
            x = x.unsqueeze(1)  # [B, 1, C, H, W]

        B, T = x.shape[:2]
        x = x.view(B * T, *x.shape[2:])  # [B*T, C, H, W]
        feats = self.backbone(x)  # [B*T, 512, H', W']
        feats = feats.view(B, T, -1)  # [B, T, 512*H'*W']

        if self.enable_temporal:
            feats = self.temporal_module(feats)  # [B, 512*H'*W']
        else:
            feats = feats[:, 0, :]  # 取第一帧

        # 后续处理...
```

## 三、三阶段训练策略

### 3.1 Stage 1：单帧强标注预训练（已完成）

| 项目 | 内容 |
|------|------|
| **目标** | 建立 WoodScape 强标注基线 |
| **数据** | Train_Real (3200) + Val_Real (800) |
| **输出** | 预训练 backbone + Tile Head + Global Head |
| **性能** | Tile MAE: 0.0396, Global MAE: 0.0187 |

### 3.2 Stage 2：引入时序模块 + SD_seq 训练

| 项目 | 内容 |
|------|------|
| **目标** | 学习"严重度变化率"与序列一致性 |
| **数据** | SD_seq 合成序列 {I'_t, Y'_t, G'_t, S'_t}_{t=1}^{T} |
| **新增损失** | L_temp = \|ΔŜ - ΔS'\| |
| **输出** | 时序感知的模型 |

### 3.3 Stage 3：混合微调（避免域偏移）

| 项目 | 内容 |
|------|------|
| **目标** | 保持强标注精度，同时获得时序能力 |
| **数据** | Train_Real (单帧) + SD_seq (时序) 按批次混合 |
| **损失** | Batch_Real: L_tile + λ·L_global<br>Batch_SD_seq: L_tile + λ·L_global + μ·L_temp |
| **输出** | 最终时序模型 |

## 四、数据需求

### 4.1 现有数据状态

| 数据类型 | 状态 | 说明 |
|----------|------|------|
| WoodScape 静态 | ✅ 已有 | 强标注，用于 Stage 1 |
| Clean 域视频 | ✅ 已有 | `dataset/my_clean_video_raw/` |
| External 视频源 | ✅ 已有 | 从视频抽帧得到 |
| SD_seq | ❌ 待生成 | 需要 Step 3 完成 |

### 4.2 SD_seq 生成需求

| 参数 | 建议值 | 说明 |
|------|--------|------|
| 序列长度 T | 5-10 | 平衡计算成本与时序信息 |
| 时间间隔 Δt | 0.1-1s | 模拟真实帧率 |
| 事件类型 | 累积/消退/突变 | 覆盖真实场景 |

### 4.3 数据隔离原则

```
┌─────────────────────────────────────────────────────────────────────┐
│                                                                     │
│  训练数据: Train_Real + SD_seq (clean 底图与 external 测试分离)        │
│  测试数据: Test_Real + Test_Ext (不参与训练)                          │
│                                                                     │
│  原则: 避免训练侧间接接触测试侧分布信息                                │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

## 五、测试集与评测

### 5.1 测试集分工

| 测试集 | 类型 | 定位 | 关键指标 |
|--------|------|------|----------|
| **Test_Real** | WoodScape 强标注 | 单帧精度基准 | Tile MAE, Global MAE, Level Acc |
| **Test_Ext** | 外部弱标注 | 时序泛化验证 | ρ(Ŝ, ext_level), ρ(Ŝ, 1-Q), 稳定性 |
| **Test_Syn** | SD_seq 合成 | 时序学习验证 | 背景一致性, Mask 平滑度, 变化率对齐 |

### 5.2 Test_Real：单帧模式推理

时序模型在 Test_Real 上采用"单帧模式"：
- 输入序列长度设为 1
- 或用同一帧复制构造长度为 K 的序列

**目的**: 验证引入时序编码模块后，静态强标注能力保持稳定

### 5.3 Test_Ext：时序收益验证

预期时序模块的收益体现在：

| 指标 | Baseline | 时序模型 | 预期变化 |
|------|----------|----------|----------|
| ρ(Ŝ, ext_level) | 0.58* | - | ↑ |
| ρ(Ŝ, 1-Q) | - | - | ↑ |
| 同视频内 Ŝ 波动 | - | - | ↓ |

*注：当前 external_test 清洗前的结果

### 7.2 完整叙事链

1. **问题提出**：
   - 现有方法基于单帧检测，易将移动物体误判为脏污
   - 脏污相对固定的物理特性未被充分利用

2. **解决方案**：
   - 设计时序编码模块，利用"固定 vs 变化"的时序对比
   - SD_seq 提供可控的时序强监督数据
   - 三阶段训练保证强标注精度与时序能力并重

3. **实验验证**：
   - Test_Real：静态强标注能力不退化
   - Test_Ext：真实视频场景泛化能力提升
   - Test_Syn：时序学习按预期工作
