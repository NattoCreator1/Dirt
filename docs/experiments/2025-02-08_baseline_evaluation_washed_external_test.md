## 一、數據清洗概述

### 1.1 清洗動機

原 external_test 數據集存在嚴重分類問題：
- Level 2 (occlusion_d) 的預測分數 (0.650) 比 Level 3 (occlusion_c) 的 (0.569) **更高**
- Spearman ρ = 0.584，相關性較低
- **根本原因**: occlusion a-e 的分類與 Severity Score 定義不一致

### 1.2 清洗策略

根據 README.md.txt，重新定義各 occlusion level：

| Level | 原定義 | 新定義 | 調整 |
|-------|--------|--------|------|
| 5 (a) | 全局嚴重遮擋 | 保持不變 | - |
| 4 (b) | 全局遮擋，較輕 | 保持不變 | - |
| 3 (c) | 部分遮擋 | 全局輕微遮擋 + 部分嚴重遮擋（中央） | **加重** |
| 2 (d) | 全局黑點 | 全局淺色 + 局部小泥點 | **減輕** |
| 1 (e) | 極輕微 | 保持不變 | - |

### 1.3 數據統計

```
總樣本數: 50,458（清洗前: 62,818）

各 level 分佈:
┌──────────┬─────────┬───────────┬─────────┐
│  Level   │  樣本數 │   比例    │ S_hat   │
├──────────┼─────────┼───────────┼─────────┤
│    5     │  3,061  │   6.1%    │  0.902  │
│    4     │ 13,752  │  27.3%    │  0.786  │
│    3     │  8,596  │  17.0%    │  0.695  │
│    2     │ 13,665  │  27.1%    │  0.615  │
│    1     │ 11,384  │  22.6%    │  0.518  │
└──────────┴─────────┴───────────┴─────────┘
```

**分佈改善**: 各 level 樣本數更加均衡（之前 level 2 佔 43.3%）
