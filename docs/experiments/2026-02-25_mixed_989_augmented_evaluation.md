# Mixed (989) + 相机域随机化 完整评估报告

**评估日期**: 2026-02-25
**实验类型**: 相机域随机化 (Camera Domain Randomization)
**目的**: 验证在线相机域随机化对跨域泛化的影响

---

## 一、实验背景

### 1.1 动机

之前的Mixed (989)实验使用WoodScape (4000) + SD (989)数据，在Test_Real上取得了优于Baseline的结果 (MAE 0.0188 vs 0.0192)，但在Test_Ext上的跨域性能没有明显提升 (ρ 0.710 vs 0.718)。

**假设**: SD合成数据只覆盖了污渍外观变化，没有覆盖相机/ISP成像链的差异。通过添加相机域随机化，可能能提升跨域泛化能力。

### 1.2 相机域随机化设计

在线数据增强，包含以下模块：

| 模块 | 概率 | 强度 |
|------|------|------|
| 颜色/曝光 (gamma, 色温, 曝光) | 0.8 | 中等 |
| 噪声 (shot noise, read noise, 色噪) | 0.7 | 中等 |
| 模糊 (defocus, motion blur) | 0.4 | 轻度 |
| 压缩伪影 (JPEG) | 0.5 | 中等 |
| 镜头效果 (vignetting) | 0.4 | 轻度 |
| 分辨率随机化 | 0.3 | 轻度 |

**结论**: 相机域随机化在同域上没有带来任何好处。

#### Test_Ext (跨域)

| 模型 | Spearman ρ | 单调性 | 等级区分 |
|------|-----------|--------|---------|
| Baseline | **0.718** | ✅ | 优秀 |
| Mixed (989) | 0.710 | ✅ | 良好 |
| Mixed + Aug | 0.669 | ❌ | 失败 |

**结论**: 相机域随机化严重损害了跨域泛化能力。

### 5.2 问题诊断

#### 为什么相机域随机化失败了？

1. **过度随机化**
   - 虽然参数设定为"中等强度"，但多个变换叠加可能引入过多噪声
   - 每个样本以高概率经历多个变换，导致信号失真

2. **视觉-标签不一致加剧**
   - Mixed (989) 已经出现轻微的保守化现象 (全等级下移)
   - 相机域随机化可能进一步加剧了视觉证据与标签之间的不匹配

3. **训练-测试分布错配**
   - 训练时使用大量域随机化
   - 测试时 (Test_Ext) 虽然有域差，但并非随机化的域差
   - 模型可能学到了错误的域不变特征

4. **高S区间失真**
   - 在严重污渍场景下，颜色/曝光等变换可能改变了污渍的视觉特征
   - 导致模型在高S区间的预测失序

### 5.3 与设计目标的差距

| 设计目标 | 预期效果 | 实际效果 |
|---------|---------|---------|
| Test_Real MAE | 保持或略优 | 0.0188 → 0.0198 (下降) |
| Test_Ext ρ | 提升到 >0.72 | 0.710 → 0.669 (下降) |
| 等级单调性 | 保持 | 破坏 |

## 六、结论与建议

### 6.1 核心结论

1. ❌ **相机域随机化实验失败**
   - 没有达到任何设计目标
   - 同域性能下降
   - 跨域性能显著下降
   - 等级单调性被破坏

2. **可能的原因**
   - 增强强度过高或概率设置不当
   - 多个变换叠加引入过多噪声
   - 视觉-标签不一致加剧

3. **关键教训**
   - 在线数据增强并非"越多越好"
   - 增强策略需要与标签特性相匹配
   - 跨域泛化提升不能仅靠域随机化

### 6.2 后续工作建议

#### 不建议继续的方向

1. ❌ 不建议继续增强当前的相机域随机化参数
2. ❌ 不建议增加更多类型的增强
3. ❌ 不建议在有问题的数据上继续实验

#### 建议尝试的方向

1. **减弱增强策略**
   - 降低所有增强的概率 (如 0.8 → 0.3)
   - 减少同时应用的变换数量
   - 仅保留最关键的变换 (如轻微噪声)

2. **改进SD数据质量**
   - 解决视觉-标签不一致问题
   - 重新审视SD生成pipeline
   - 考虑使用alpha形态软化策略

3. **其他跨域泛化方法**
   - 尝试域自适应训练
   - 使用对比学习
   - 探索时序信息利用
