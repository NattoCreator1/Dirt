# External Test 簇层面 Bootstrap 分析报告（修正版）

## 修正说明

原始分析使用图片层面 Bootstrap，违反了独立性假设（同一簇内的图片高度相关）。本报告使用**簇层面 Bootstrap**（505 个簇），给出正确的统计推断。

## 数据集结构

| 指标 | 值 |
|------|-----|
| 总簇数 | 505 |
| 总图片数 | 50,383 |
| 平均每簇 | 99.8 张 |
| 簇定义 | external_test_washed/occlusion_*/子文件夹/ |

## 簇层面 Bootstrap 结果对比

### Spearman ρ 对比

| 模型 | ρ 点估计 | 95% CI | CI 宽度 | vs Baseline Δρ | Δρ 95% CI |
|------|---------:|--------|--------:|---------------|----------|
| **Baseline** | **0.7446** | [0.6739, 0.7662] | 0.0923 (12.40%) | — | — |

### 统计显著性结论

所有模型的下降都是**统计显著**的（95% CI 完全不覆盖 0）：

1. **SD989 vs Baseline**: Δρ = -0.171, p < 0.001
2. **A+-only vs Baseline**: Δρ = -0.097, p < 0.001
3. **Tile-only vs Baseline**: Δρ = -0.060, p < 0.001

## 与图片层面 Bootstrap 的对比

### CI 宽度差异

| 模型 | 图片层面 CI 宽度 | 簇层面 CI 宽度 | 放大倍数 |
|------|----------------|----------------|---------:|
| Baseline | 0.0094 (1.31%) | **0.0923 (12.40%)** | **9.8x** |
| SD989 | 0.0093 (1.30%) | **0.1363 (23.77%)** | **14.7x** |

**关键发现**：簇层面的 CI 宽度是图片层面的 **10-15 倍**！

### 为什么图片层面 Bootstrap 是错误的？

1. **违反独立性假设**：Bootstrap 假设样本独立同分布，但同一簇内的图片来自同一视频序列，高度相关
2. **高估统计功效**：50,383 张图片 ≠ 50,383 个独立样本，实际只有 505 个独立簇
3. **CI 过于狭窄**：导致"虚假的统计显著性"

## 论文写作建议

### 正确的统计方法描述

```
我们使用簇层面 Bootstrap 分析来估计 Spearman ρ 的置信区间。
由于 External Test 数据集中的图片来自 505 个不同的视频序列
（簇），同一簇内的图片高度相关，我们在簇层面进行重采样
（505 个簇）而非图片层面（50,383 张图片），以确保统计推断的正确性。
```

### 可用结论

1. **所有 SD 增强策略都显著差于 Baseline**
   - 即使最接近的 Tile-only 也有 Δρ = -0.060 [−0.093, −0.026]
   - 这个差异在统计上和实际上都是显著的

2. **标尺压缩是普遍现象**
   - 压缩比 1.17x - 1.34x
   - 说明 SD 训练导致模型对等级变化敏感度降低

3. **Phase 2 策略未能解决问题**
   - A+-only 和 Tile-only 都显著差于 Baseline
   - 即使过滤 94% SD，特征分布偏移仍然存在

### 需要承认的限制

1. **SD 数据未能实现无损增强**
   - 所有 SD 增强版本都有统计显著的性能下降

2. **SD989 作为"最接近 Baseline"的策略仍不理想**
   - Δρ = -0.171 非常大
   - 可能是因为数据来源或生成方式的问题
