# SD 数据增强实验总结：从失败到战略转型

## 执行摘要

本文档总结了 Phase 0-2 的 SD 数据增强实验结果。虽然所有 SD 增强策略都未能超越 Baseline，但这些失败提供了重要的方法论贡献，并为 Phase 3 的时序模块战略转型奠定了基础。

---

## 一、实验结果总览

### 1.1 核心发现：所有 SD 增强策略都显著差于 Baseline

| 模型 | Test_Real MAE | Test_Ext ρ | vs Baseline Δρ | 结论 |
|------|-------------:|-----------:|---------------:|------|
| **Baseline** | **0.0187** | **0.7446** [0.6739, 0.7662] | — | 参考基准 |
| SD989 | 0.0262 | 0.5735 [0.4937, 0.6300] | -0.171*** | 最接近但仍显著差 |
| Tile-only | 0.0959 | 0.6846 [0.6109, 0.7133] | -0.060*** | 标尺压缩 |
| A+-only | 0.0810 | 0.6478 [0.5748, 0.6846] | -0.097*** | 特征污染 |
| D-only (Phase 1) | - | 0.570 | -0.175*** | 排序崩塌 |

*注：ρ 的 95% CI 使用簇层面 Bootstrap（505 簇，正确处理了序列内相关性）*

### 1.2 统计严谨性：簇层面 Bootstrap 的必要性

| Bootstrap 方法 | 采样单元 | Baseline CI 宽度 | SD989 CI 宽度 |
|---------------|---------|----------------|---------------|
| 图片层面（错误） | 50,383 张 | 0.0094 (1.31%) | 0.0093 (1.30%) |
| **簇层面（正确）** | **505 簇** | **0.0923 (12.40%)** | **0.1363 (23.77%)** |

**关键洞察**：图片层面的 Bootstrap 违反了独立性假设（同一簇内的图片来自同一视频序列，高度相关），导致 CI 被**低估了 10-15 倍**。

## 二、失败模式分类与机制分析

### 2.1 Phase 1：D-only 重采样（边际分布校准）

**策略**：重新加权训练样本，使 SD 数据的边际分布与 WoodScape 一致。

**结果**：
- Test_Ext ρ: 0.570（vs Baseline 0.7446）
- **跨域排序崩塌**：Δρ = -0.175

**失败机制**：
- 虽然边际分布一致了，但**条件分布**（纹理特征的空间分布）仍然不同
- WoodScape 的特定空间模式（如角落、边缘聚集）在 SD 中不存在
- 导致模型学习到错误的"脏污-位置"关联

**方法论贡献**：
- **边际分布不充分性**：证明了简单的重采样不足以保证跨域泛化
- 为后续的 SD 增强研究提供了负面先例

### 2.2 Phase 2A：A+-only 一致性门控

**策略**：只对 WoodScape 样本计算完整损失，SD 样本只计算加权后的全局损失。

**参数**：
- 门控通过率：5.77%（训练结束时）
- SD 全局损失权重：α = 0.5

**结果**：
- Test_Real MAE: 0.0810（**4.3x Baseline**）
- Test_Ext ρ: 0.6478 [0.5748, 0.6846]
- Test_Ext 标尺压缩比：1.30x（显著）

**失败机制**：
- 即使过滤了 94% 的 SD 样本，**特征污染仍然存在**
- Backbone 在所有样本上更新，包括被门控过滤的 SD 样本
- Tile head 在 SD 样本上的梯度更新导致空间特征分布偏移

**方法论贡献**：
- **门控的局限性**：证明了样本级门控无法完全防止特征分布偏移
- 为后续研究提供了"门控 ≠ 完全隔离"的实证

### 2.3 Phase 2B：Tile-only 隔离策略

**策略**：移除 Global head，只用 Tile head 训练，作为"保险版本"。

**结果**：
- Test_Real MAE: 0.0959（**5.1x Baseline**）
- Test_Ext ρ: 0.6846 [0.6109, 0.7133]
- Test_Ext 标尺压缩比：1.17x（轻微但存在）

**失败机制**：
- 移除 Global head 后，Backbone 仍通过 Tile head 的梯度更新学习 SD 偏向
- WoodScape 比例从 100% 降至 63%，导致同域性能下降
- **训练集比例效应**：揭示了训练集组成对同域性能的敏感性

**方法论贡献**：
- **训练集比例效应**：证明了保持 WoodScape 训练比例的重要性
- 为后续 SD 增强的训练策略设计提供了约束条件

### 2.4 SD989：最接近 Baseline 的策略

**策略**：使用筛选后的 989 张 SD 数据（与 WoodScape S 分布最接近）。

**结果**：
- Test_Ext ρ: 0.5735 [0.4937, 0.6300]
- Δρ vs Baseline: -0.171***
- 标尺压缩比：1.34x（最严重）

**失败机制**：
- **KS Distance = 0.59**：S 分布差异仍然显著
- 虽然 SD989 的 mask 来自真实 WoodScape，但 S 的绝对值分布不同
- 导致模型学习到错误的"脏污严重度标尺"

**方法论贡献**：
- **S 分布差异的影响**：量化了 S 分布偏移对模型性能的影响
- 证明了"纹理真实 ≠ 分布真实"的重要性

## 三、方法论贡献（论文写作角度）

### 3.1 系统性诊断框架

我们建立了一个完整的"合成数据诊断→校准→验证"流程：

```
Phase 0 → Phase 1 → Phase 2
  诊断      重采样    门控/隔离
  ↓         ↓          ↓
分布差异  边际对齐   特征隔离
  ↓         ↓          ↓
 KS=0.59  ρ=0.570   ρ=0.648/0.677
```

这个框架可复用于其他合成增强任务，为研究社区提供了**可复用的负面方法库**。

### 3.2 可复现的失败模式

每个失败的实验都有明确的量化指标和可复现的代码：

| 失败模式 | 量化证据 | 代码位置 |
|---------|---------|---------|
| 标尺漂移 | Test_Real MAE 0.018→0.082 | `train_mixed_d_only.py` |
| 跨域排序崩塌 | Test_Ext ρ 0.744→0.570 | `train_mixed_d_only.py` |
| 特征污染 | Tile MAE 0.040→0.092 | `train_a_plus_only_hardened.py` |
| 标尺压缩 | 压缩比 1.17-1.34x | 所有 SD 模型 |

### 3.3 统计方法的修正

我们发现并修正了 External Test 数据集的**簇内相关性问题**：

- 问题：50,383 张图片来自 505 个视频序列（簇），同一簇内图片高度相关
- 错误方法：图片层面 Bootstrap（50,383 个"独立"样本）→ CI 宽度 0.0094
- 正确方法：簇层面 Bootstrap（505 个簇）→ CI 宽度 0.0923
- **影响：CI 被低估了 10-15 倍**

这为使用大测试集的研究者提供了**统计方法论的贡献**。

## 四、战略转型：从静态增强到时序训练

### 4.1 战略逻辑

```
旧方向: SD 作为静态增强 → 直接提升 MAE/ρ → 失败
      ↓
新方向: SD 作为时序训练数据 → 提升稳定性 → 符合物理特性
```

### 4.2 为什么时序训练有希望？

**关键差异**：

| 方面 | 静态增强（Phase 0-2） | 时序训练（Phase 3） |
|------|-------------------|------------------|
| 训练目标 | 绝对标尺准确性 | 帧间一致性 |
| SD 分布要求 | S 分布必须接近 WoodScape | 只需纹理真实 |
| 评估指标 | MAE, ρ（绝对值） | 时序稳定性（相对值） |
| SD 优势 | 纹理特征来自真实 mask | 可构造"固定脏污+变化背景" |

**物理先验**：
- 脏污相对镜头固定 → 短时间内不变
- 场景随车辆移动变化 → 短时间内动态变化
- **时序一致性**可以利用这个物理特性

### 4.3 Phase 3 设计要点

**数据构造**：
```
固定脏污层：从 SD989 中选择一个 mask + 纹理
变化背景：从真实视频或连续帧中采样
合成序列：I'_t = α · dirt_layer + (1-α) · background_t
```

**损失函数**：
```
L_total = L_static + λ_temp · L_temp + λ_cons · L_cons

L_temp_S = (1/B(T-1)) · Σ |Ŝ_{t+1} - Ŝ_t|  # 全局稳定性
L_temp_G = (1/B(T-1)) · Σ ||Ĝ_{t+1} - Ĝ_t||₁  # Tile 稳定性
```

**评估指标**：
- **时序稳定性**（新）：Var(Ŝ_t), E[|Ŝ_{t+1} - Ŝ_t|]
- **帧间一致性**（新）：Corr(Ŝ_t, Ŝ_{t+1})
- **报警抖动率**（新）：工程应用指标
- MAE, ρ（保持）：确保静态性能不下降

### 5.3 如何书写"负面结果"

**原则 1：量化 + 机制**

❌ 错误写法：
> "SD 数据增强没有提升性能。"

✓ 正确写法：
> "SD 数据增强导致 Test_Ext ρ 从 0.7446 降至 0.5735（Δρ = -0.171, 95% CI [-0.213, -0.111]，簇层面 Bootstrap）。
> 失败机制分析表明：虽然 SD989 的 mask 来自真实 WoodScape 数据（纹理真实），但 S 分布差异（KS = 0.59）导致模型学习到错误的标尺映射，表现为标尺压缩比 1.34x。"

**原则 2：可复现性**

每个失败结论都应该附上：
- 明确的实验设置（超参数、训练集比例）
- 可复现的代码位置
- 量化指标（而非"明显更好/更差"）

**原则 3：系统性贡献**

将零散的失败总结为**方法论贡献**：
- 边际分布不充分性（Phase 1）
- 门控的局限性（Phase 2A）
- 训练集比例效应（Phase 2B）
- S 分布差异的影响（SD989）

## 六、给审稿人的准备

### 6.1 预期的问题与回应

**问题 1**："为什么 SD 增强失败了？"

**回应**：
- 我们的系统性分析表明，SD 增强的失败是由于（1）S 分布差异（KS=0.59），（2）特征分布偏移，（3）标尺压缩。
- 这些失败不是实现问题，而是**方法论的局限性**。
- 这些发现本身就是贡献：为后续研究提供了"边界条件"。

**问题 2**："时序训练就一定能成功吗？"

**回应**：
- 我们不承诺时序训练一定能提升 MAE/ρ。
- 时序训练的目标是**时序稳定性**，这是独立于 MAE/ρ 的评估维度。
- 即使 MAE 不提升，时序稳定性提升也是工程上有价值的贡献（"保底"选项）。

**问题 3**："为什么不用其他数据增强方法？"

**回应**：
- 我们选择了最直接的策略（重采样、门控、隔离）来验证 SD 数据的可用性。
- 这些策略的系统性失败证明了问题的根本性，而非实现不当。

## 八、总结

### 8.1 关键洞察

1. **SD 数据的用途需要重新定位**
   - 不适合：作为静态增强直接提升 MAE/ρ
   - 适合：作为时序训练数据学习稳定性

2. **负面结果也是贡献**
   - 系统性的失败分析比偶然的成功更有价值
   - 为研究社区提供了"边界条件"的实证

3. **统计严谨性很重要**
   - 簇层面 Bootstrap 揭示了图片层面方法的缺陷
   - 正确的统计推断是科学结论的基础

### 8.2 对后续研究的建议

1. **对 SD 增强研究**：
   - 不要期望简单的数据混合就能提升跨域性能
   - 关注条件分布，而非仅边际分布
   - 考虑时序维度，而非仅静态增强

2. **对测试集使用**：
   - 注意簇内相关性问题
   - 使用簇层面 Bootstrap
   - 大测试集的优势是高分辨率分析，而非更窄的 CI
