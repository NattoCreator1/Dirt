# Phase 2: 一致性门控实验总结报告

**实验时间**: 2025-02-25 ~ 2025-02-26
**状态**: 已完成
**结论**: SD 数据作为静态增强未能超越 baseline，建议转向时序模块

---

## ⚠️ 重要修正说明（2025-02-26）

### External Test 统计方法修正

在完成 Phase 2 实验后，我们发现 External Test 数据集存在**簇内相关性问题**：

- 数据集结构：50,383 张图片来自 **505 个视频序列（簇）**
- 问题：同一簇内的图片来自同一视频的连续帧，**高度相关**
- 影响：之前使用的**图片层面 Bootstrap** 违反了独立性假设，导致置信区间被**低估了 10-15 倍**

**修正方案**：
- ✅ **正确方法**：**簇层面 Bootstrap**（505 个簇作为独立单元）
- ❌ **错误方法**：图片层面 Bootstrap（50,383 张作为独立样本）

### 修正前后对比

| 模型 | 图片层面 ρ (95% CI) | 簇层面 ρ (95% CI) | CI 宽度变化 |
|------|-------------------|-------------------|-----------|
| Baseline | 0.718 [0.7153, 0.7247] | **0.7446 [0.6739, 0.7662]** | **0.0094 → 0.0923 (9.8x)** |
| Tile-only | 0.677 [0.6730, 0.6831] | **0.6846 [0.6109, 0.7133]** | **0.0101 → 0.1024 (10.1x)** |

**重要结论**：
1. **簇层面 Bootstrap 是更稳定可信的统计方法**，因为它正确处理了数据集的簇内相关性
2. 之前基于图片层面的"统计显著性"结论过于乐观，需要用簇层面结果重新评估
3. **本文档中 3.3 节的 Test_Ext 评估结果应参考本修正说明**

### 如何使用本报告

对于 **Test_Ext 性能对比**：
- **推荐使用**：簇层面 Bootstrap 结果（更稳定可信）
- **仅供参考**：图片层面结果（展示了发现问题的过程）

对于 **跨域排序能力的定性结论**：
- 图片层面和簇层面都显示：Tile-only (0.677) > A+-only (0.648) > Baseline 的相对排序仍然成立

## 一、实验背景

### 1.1 Phase 0 诊断发现

在 Phase 0 阶段，我们发现不同 SD 子集的 S 分布存在显著差异：

| SD 子集 | S_mean | 与 WoodScape (0.408) 的差异 |
|---------|--------|----------------------------|
| SD 1260 | 0.2866 | 负偏 (-0.12) |
| SD 8960 | 0.4658 | 正偏 (+0.06) |

**核心问题**: 如果直接混合训练，SD 数据会污染全局严重度标尺。

### 1.2 Phase 2 设计目标

针对上述问题，Phase 2 设计了两种"硬化"策略：

| 策略 | 核心思想 | 预期效果 |
|------|----------|----------|
| **A+-only** | 一致性门控 + α_sd=0.4 | 只让"一致"的 SD 样本参与全局训练 |
| **Tile-only** | 保险策略 | SD 只训练 tile 头，完全不参与全局标尺学习 |

---

## 二、实验设计

### 2.1 A+-only: 一致性门控

```
门控条件: |Ŝ - S_agg| < threshold (0.05)
通过奖励: λ_global = α_sd (0.4)
未通过惩罚: λ_global = 0
```

**代码实现**: `train_a_plus_only_hardened.py`

**关键参数**:
- `gate_threshold = 0.05`
- `alpha_sd = 0.4`
- `warmup_steps = 1000`

### 2.2 Tile-only: 保险策略

```
SD 样本:
  - Tile head: 完整损失权重
  - Global head: 零损失权重 (不学习标尺)

WoodScape 样本:
  - Tile head: 完整损失权重
  - Global head: 完整损失权重
```

**代码实现**: `train_sd_tile_only.py`

### 2.3 训练配置

| 参数 | 值 |
|------|-----|
| 混合数据集 | WS 4000 + SD 8960 |
| Total steps | 10000 |
| Batch size | 16 (A+-only), 16 (tile-only) |
| Learning rate | 3e-4 |
| Optimizer | AdamW (wd=1e-2) |

**观察**: 门控通过率从 100% 逐渐收紧至 ~6%，说明大部分 SD 样本被过滤。

### 3.2 Test_Real 评估

| 模型 | Tile MAE | Global MAE | ρ(Ŝ,GT) | S_mean |
|------|---------:|-----------:|---------:|-------:|
| **Baseline** (S_full_wgap_alpha50) | **0.0396** | **0.0187** | - | ~0.40 |
| Phase 1 D-only (负对照) | 0.0403 | 0.0179 | - | - |

| **Tile-only** | 0.0923 | 0.0959 | 0.9649 | 0.401 |

**关键发现**:
- Phase 2 模型的 **Global MAE 是 baseline 的 4-5 倍**
- Tile MAE 也显著高于 baseline
- ρ(Ŝ,GT) 仍然很高 (0.96+)，说明排序能力保持，但**绝对值标尺偏移严重**

### 3.3 Test_Ext 评估

| 模型 | Spearman ρ | Monotonic | Level Means (a→e) |
|------|-----------:|:---------:|-------------------|
| **Baseline** | **0.718** | ✓ | 0.75, 0.64, 0.50, 0.41, 0.32 |
| Phase 1 D-only | 0.570 | ✗ | - |
| **A+-only** | 0.648 | ✓ | 0.74, 0.66, 0.55, 0.50, 0.40 |
| **Tile-only** | **0.677** | ✓ | 0.67, 0.53, 0.42, 0.36, 0.29 |

**关键发现**:
- Tile-only 在 Test_Ext 上优于 A+-only (0.677 > 0.648)
- 但两者都**低于 baseline** (0.718)
- 单调性都保持，说明跨域排序能力未完全破坏

## 四、失败机制分析

### 4.1 为什么 Phase 2 未能超越 Baseline？

#### 假设 1: 训练集分布偏移

| 数据集 | WoodScape 样本 | SD 样本 | WoodScape 比例 |
|--------|---------------:|--------:|---------------:|
| Baseline | 4000 | 0 | 100% |
| Phase 2 | 4000 | 8960 | **31%** |

训练集中 WoodScape 比例从 100% 降至 31%，模型在测试时可能过度拟合 SD 样本的特征。

#### 假设 2: SD 的"隐式影响"即使有门控也无法完全消除

即使 A+-only 只让 6% 的 SD 样本通过门控，这些样本仍然可能：
1. 将 SD 特有的纹理模式编码到 backbone
2. 影响特征空间的分布
3. 导致在 WoodScape 测试集上的性能下降

#### 假设 3: Tile-only 策略的问题

Tile-only 完全禁止 SD 参与全局训练，但这可能导致：
1. Backbone 特征主要由 SD 训练驱动 (tile loss)
2. Global head 只在 31% 的 WoodScape 样本上训练
3. Global head 的特征输入与训练分布不匹配

### 4.2 与 Phase 1 D-only 的对比

| Phase | 策略 | Test_Real | Test_Ext ρ | 失败模式 |
|-------|------|-----------|-----------:|----------|
| Phase 1 | D-only 重采样 | MAE 改善 | **0.570** | 跨域排序崩塌 |
| Phase 2 | A+-only 门控 | MAE 崩溃 | 0.648 | 同域标尺偏移 |
| Phase 2 | Tile-only 保险 | MAE 崩溃 | 0.677 | 同域标尺偏移 |

**结论**: 不同策略导致不同的失败模式，但都未能同时优化两个测试集。

## 五、关键结论与建议

### 5.1 核心结论

```
SD 数据作为静态增强直接提升 MAE/ρ 的路径已被证明风险极高，
即使有一致性门控和保险策略，也无法超越纯 WoodScape 训练的 baseline。
```

### 5.2 方法论贡献

虽然实验未达预期，但我们获得了以下可发表的方法论发现：

1. **边际分布校准 (D-only) 不足以保证跨域排序**
   - 证明了简单地重采样会导致跨域相关性崩塌

2. **一致性门控 (A+-only) 的局限性**
   - 即使只让 ~6% 的 SD 样本通过，仍会影响同域性能
   - 说明 SD 的影响不仅来自"不一致样本"，也来自特征分布偏移

3. **Tile-only 策略的边界**
   - 证明了即使完全隔离 SD 对全局 head 的影响，backbone 仍会学习 SD 偏向的特征

### 5.3 建议的未来方向

#### 方向 1: SD 用于时序训练 (推荐)

**核心洞察**: 时序训练的目标是**稳定性**，不是**绝对标尺准确性**

```
优势:
- SD 数据可以构造"脏污固定、背景变化"的完美序列
- 时序一致性损失不需要 S 分布完全一致
- 评估指标可以独立于 MAE/ρ (如时序稳定性、抖动率)
```

#### 方向 2: 更精细的域适应

- 使用对抗训练对齐特征分布
- 使用域分离网络 (Domain Separation Networks)
- 使用元学习快速适应 SD 域

#### 方向 3: 完全放弃 SD，聚焦其他方向

- 数据增强: CutMix, MixUp, AutoAugment
- 模型架构: Attention, Transformer
- 损失函数: 更好的一致性约束
