## 一、核心洞察与动机

### 1.1 物理观察：脏污 vs 场景的时序特性

```
┌─────────────────────────────────────────────────────────────────────┐
│                                                                     │
│  镜头脏污 (Dirt):  相对镜头固定 → 随时间近似静止                      │
│  道路场景 (Scene): 随车辆移动而变化 → 随时间动态变化                   │
│                                                                     │
│  → 通过"固定 vs 变化"的时序对比，可以更准确地识别脏污                    │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

**关键假设**:
- 短时间内 (如 1-5 秒)，镜头上的脏污位置、形状、严重度基本不变
- 同一时间段内，背景场景会随着车辆移动发生显著变化

### 1.2 从 Phase 2 失败到 Phase 3 的战略转型

#### Phase 2 的核心发现 (经验贡献)

虽然 Phase 2 的 SD 静态增强策略**未能超越 baseline**，但为后续工作提供了关键经验：

| Phase 2 策略 | Test_Real MAE | Test_Ext ρ | 关键发现 |
|-------------|--------------:|-----------:|----------|
| Baseline | 0.0187 | 0.718 | 参考基准 |
| D-only | 0.0179 | **0.570** | 边际分布校准 → 跨域排序崩塌 |
| A+-only | 0.0810 | 0.648 | 门控过滤 94% SD → 仍有特征污染 |
| Tile-only | 0.0959 | 0.677 | 隔离全局头 → Backbone 仍学习 SD 偏向 |

**方法论贡献** (可写入论文):
1. **边际分布不充分性**: 证明了简单的重采样 (D-only) 不足以保证跨域泛化
2. **门控的局限性**: 一致性门控即使过滤 94% 的 SD 样本，仍无法完全消除特征分布偏移
3. **训练集比例效应**: WoodScape 比例从 100% 降至 31% 导致同域性能下降 4-5 倍

#### 战略转型的逻辑

```
旧方向: SD 作为静态增强 → 直接提升 MAE/ρ → 失败
      ↓
新方向: SD 作为时序训练数据 → 提升稳定性 → 更符合物理特性
```

**核心洞察**: 时序训练的目标是**帧间一致性**，而非**绝对标尺准确性**

这意味着:
- SD 数据的 S 分布差异 (KS=0.59) 不再是致命问题
- 关键是 SD 的**纹理特征**来自真实 WoodScape mask，空间分布真实
- 可以通过"固定脏污层 + 变化背景"构造物理一致的时序序列

### 1.3 SD989 作为时序数据底座的可行性验证

#### 分布差异分析

| 数据集 | N | S_mean | S_std | KS Distance (vs WS) |
|--------| -: | ------: | -----: | -------------------: |
| WoodScape Test | 1000 | 0.4924 | 0.2751 | - |
| SD989 | 989 | **0.2672** | **0.0824** | **0.5908*** |

*KS=0.59 表示分布差异显著，但这不影响时序训练

#### 为什么分布差异不影响时序训练？

1. **纹理真实**: SD989 的 mask 来自真实 WoodScape 数据，脏污的空间分布、遮挡模式都是真实的
2. **时序目标不同**: 我们要学习的是"同一块脏污在连续帧上预测应该稳定"，不需要 S 的绝对值准确
3. **混合训练策略**: 真实帧学习标尺，SD 序列学习稳定性，两者互补

## 二、数理分析与理论框架

### 2.1 时序一致性问题的数学建模

#### 单帧模型的局限性

对于单帧模型，预测可以表示为:

```
Ŝ_t = f(I_t; θ)
```

其中:
- `I_t` 是第 t 帧图像
- `θ` 是模型参数
- `Ŝ_t` 是预测的严重度

**问题**: 单帧模型每帧独立预测，没有利用时序信息。

#### 时序模型的优势

对于时序模型，预测可以表示为:

```
Ŝ_t = f(I_{t-T}, ..., I_{t-1}, I_t; θ, θ_temp)
```

其中:
- `I_{t-T:t}` 是 T 帧序列
- `θ_temp` 是时序模块参数

**优势**: 模型可以学习"脏污静止、场景变化"的物理先验。

### 2.2 物理先验的形式化

#### 假设: 脏污严重度在短时间内不变

对于连续帧序列 `{I_t}_{t=1}^T`，如果脏污固定，则:

```
S_true(t) ≈ S_true(t+1), ∀ t ∈ [1, T-1]
```

#### 推论: 模型预测应该稳定

```
L_temp = (1/(T-1)) · Σ_{t=1}^{T-1} |Ŝ_t - Ŝ_{t+1}|
```

这个损失不需要强标注，只需要"脏污基本不变"的假设成立。

### 2.3 三种时序建模方式

| 方式 | 数学形式 | 计算复杂度 | 适用场景 |
|------|---------|-----------|---------|
| **LSTM/GRU** | h_t = GRU(h_{t-1}, f_t) | O(T×H) | 长序列建模 |
| **Temporal Conv** | h_t = Conv(f_{t-k:t}) | O(k×H) | 实时应用 |
| **Temporal Attn** | h_t = Σ α_{ti} f_i | O(T²×H) | 需要可解释性 |

其中 `f_t` 是第 t 帧的 backbone 特征，`h_t` 是时序聚合特征。

**推荐方案**: Temporal Conv (1D卷积) + 可选 Attention

## 三、实验设计

### 3.1 整体架构

```
┌─────────────────────────────────────────────────────────────────────┐
│                        时序模型架构                                   │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   输入: [I_{t-T+1}, ..., I_{t-1}, I_t]  (T 帧序列)                  │
│                        ↓                                            │
│   共享 Backbone (ResNet18)                                          │
│                        ↓                                            │
│   特征序列: [F_{t-T+1}, ..., F_{t-1}, F_t]  ∈ R^{T×512×H×W}        │
│                        ↓                                            │
│   ┌────────────────────────────────────────────────────┐             │
│   │            时序编码模块 (Phase 3 新增)               │             │
│   │  ┌────────────────┐    ┌────────────────┐          │             │
│   │  │ Temporal Conv  │ →  │  Temporal Attn │ (可选)    │             │
│   │  │  (1D Conv, k=3) │    │  (Self-Attn)   │          │             │
│   │  └────────────────┘    └────────────────┘          │             │
│   └────────────────────────────────────────────────────┘             │
│                        ↓                                            │
│   时序感知特征: F_temporal ∈ R^{512×H×W}                            │
│                        ↓                                            │
│   ┌────────────────┴────────────────┐                              │
│   ↓                                 ↓                              │
│ Tile Head                         Global Head                       │
│ Ĝ [B,4,8,8]                       Ŝ [B,1]                          │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

### 3.2 数据构造策略

#### 策略 1: SD 序列 (推荐用于验证)

```
固定脏污层: 从 SD989 中选择一个 mask + 纹理
变化背景: 从真实视频或连续帧中采样
合成序列: I'_t = α · dirt_layer + (1-α) · background_t
```

**优势**:
- 完美控制"脏污固定、背景变化"
- 天然满足时序一致性假设
- 可以大规模生成

#### 策略 2: 真实 WoodScape 序列

```
从 WoodScape 数据集中选择同一相机的连续帧
由于拍摄间隔短，脏污基本不变
```

**优势**:
- 真实场景
- 包含自然的变化模式

**劣势**:
- 数量有限
- 可能有少量脏污变化

#### 策略 3: 混合训练

```
阶段 1: 在真实数据上预训练单帧模型 (已完成: baseline)
阶段 2: 在 SD 序列上训练时序模块
阶段 3: 混合真实序列 + SD 序列微调
```

### 3.3 损失函数设计

#### 总损失

```
L_total = L_static + λ_temp · L_temp + λ_cons · L_cons
```

其中:
- `L_static` 是单帧监督损失 (tile + global)
- `L_temp` 是时序一致性损失
- `L_cons` 是预测一致性损失 (Ŝ vs S_agg)

#### 时序一致性损失

```
L_temp = L_temp_S + μ · L_temp_G

L_temp_S = (1/B(T-1)) · Σ_{b,t} |Ŝ_{b,t+1} - Ŝ_{b,t}|

L_temp_G = (1/B(T-1)) · Σ_{b,t} ||Ĝ_{b,t+1} - Ĝ_{b,t}||₁
```

**物理含义**:
- `L_temp_S`: 全局严重度在连续帧上应该平滑
- `L_temp_G`: Tile 预测在连续帧上应该平滑

#### 预测一致性损失

```
L_cons = (1/B) · Σ_b |Ŝ_b - S_agg_b|
```

这是 baseline 中已有的损失，继续保留。

### 3.4 评估指标设计

#### 传统指标 (与 baseline 一致)

| 指标 | 说明 |
|------|------|
| Tile MAE | Tile 预测的平均绝对误差 |
| Global MAE | 全局严重度 MAE |
| Spearman ρ (Test_Ext) | 跨域排序相关性 |

#### 新增时序指标

| 指标 | 数学形式 | 物理含义 |
|------|---------|----------|
| **时序稳定性** | Var(Ŝ_t) 或 E[\|Ŝ_{t+1} - Ŝ_t\|] | 预测的抖动程度 |
| **时序单调性** | 违反单调性的帧占比 | 是否出现剧烈跳变 |
| **帧间一致性** | Corr(Ŝ_t, Ŝ_{t+1}) | 相邻帧预测的相关性 |

#### 工程意义指标

| 指标 | 应用场景 |
|------|----------|
| **报警抖动率** | 实际部署中，频繁的报警开关会造成用户体验差 |
| **误报持续时间** | 错误报警持续的平均时长 |
| **响应延迟** | 脏污出现到正确检测的时间 |

## 四、实验阶段规划

### 4.1 Phase 3A: 时序模块验证 (简化版)

**目标**: 验证时序模块能否提升稳定性

**数据**:
- SD989 序列 (固定脏污 + 变化背景)
- 真实 WoodScape 序列 (少量)

**模型**:
- Baseline + Temporal Conv (k=3, T=5)

**训练**:
- 固定 backbone，只训练时序模块
- 损失: L_temp only

**评估**:
- 时序稳定性指标
- Test_Real/Test_Ext 性能

**预期结果**:
- 时序稳定性显著提升 (抖动减少)
- MAE/ρ 可能持平或小幅提升

### 4.2 Phase 3B: 端到端训练

**目标**: 联合优化静态监督 + 时序一致性

**数据**:
- 混合数据集: WoodScape + SD 序列

**模型**:
- 完整时序模型
- 端到端训练

**训练**:
- 损失: L_total = L_static + L_temp + L_cons

**评估**:
- 完整指标集

### 4.3 Phase 3C: 消融实验

| 实验 | 时序模块 | L_temp | L_cons | 目的 |
|------|---------|--------|--------|------|
| Baseline | ✗ | ✗ | ✓ | 参考基准 |
| + Temporal Conv | ✓ | ✗ | ✓ | 验证时序编码 |
| + L_temp | ✓ | ✓ | ✓ | 验证时序损失 |
| + Attention | ✓(Attn) | ✓ | ✓ | 验证 Attention |

## 五、SD 工作的经验贡献总结

### 5.1 方法论贡献

虽然 Phase 2 的 SD 静态增强策略**未能超越 baseline**，但为后续工作提供了关键经验：

#### 贡献 1: 系统性诊断框架

```
Phase 0 → Phase 1 → Phase 2
  ↓         ↓         ↓
诊断      重采样    门控/隔离
```

我们建立了一个完整的"合成数据诊断→校准→验证"流程，可复用于其他合成增强任务。

#### 贡献 2: 失败机制的可复现性

每个失败的实验都有明确的量化指标和可复现的代码：

| 失败模式 | 量化证据 | 代码位置 |
|---------|---------|---------|
| 标尺漂移 | Test_Real MAE 0.018→0.082 | `train_a_plus_only_hardened.py` |
| 跨域排序崩塌 | Test_Ext ρ 0.718→0.570 | `train_mixed_d_only.py` |
| 特征污染 | Tile MAE 0.040→0.092 | `train_sd_tile_only.py` |

#### 贡献 3: 边界条件的明确

我们明确了合成数据使用的边界条件：

```
✗ 适合: 直接混合训练提升 MAE/ρ
✓ 适合: 时序一致性训练 (目标不同)
✓ 适合: 数据增强的辅助手段
```

### 5.2 对后续工作的指导意义

#### 指导 1: 选择合适的评估目标

合成数据不适合作为"提升绝对标尺准确性"的手段，但适合作为"提升稳定性/一致性"的手段。

#### 指导 2: 训练策略的调整

```
静态增强: 混合训练 → 标尺污染 → 失败
时序增强: 分别训练 → 特征互补 → 可能成功
```

#### 指导 3: 评估指标的独立性

时序稳定性是一个**独立于 MAE/ρ** 的指标：
- 即使 MAE 不提升，时序稳定性提升也是工程上有价值的贡献
- 这为论文贡献提供了"保底"选项
