**创建日期**: 2026-02-26
**基于**: Phase 2 失败分析 + 战略转型讨论
**核心定位**: SD 从"静态增强工具"转型为"时序/稳定性数据构造工具"

## 一、研究背景与动机

### 1.1 Phase 2 的核心发现（负结果的经验贡献）

**发现 1：SD 在严重度标尺任务中的多重失效机制**

SD 合成数据在精细严重度回归任务中的失效，**不只**是"生成脏污与 mask/严重程度不匹配"，而是更复杂的系统性问题：

| 失效机制 | 具体表现 | 证据 |
|---------|---------|------|
| **视觉机制—标签语义不一致** | 合成图像的视觉退化与标签 S 之间存在偏差 | Phase 1 D-only 校准 S 分布后仍崩塌 |
| **合成域捷径特征** | 模型学到 SD 特有的纹理/噪声/频谱伪影，而非真实退化机制 | SD989 标尺压缩比 1.34x |
| **成像链统计不一致** | 外域相机/ISP 与 SD 合成统计不同，跨域排序不稳 | 所有 SD 模型 Test_Ext ρ 显著下降 |
| **标尺语义漂移** | 即使 S 定义一致，合成样本对同一 S 的"视觉证据强度"不同 | 压缩比 1.17x - 1.34x |

**关键结论**：
> SD 合成在严重度标尺任务中存在"视觉退化机制—标签语义"不一致以及**合成域特有的捷径特征**，这是导致跨域排序能力下降的根本原因。边际分布对齐（D-only）无法修复这些问题。

**发现 2：边际分布对齐不足以修复标尺漂移**
- Phase 1 D-only 重采样实验: 对 SD8960 做 importance sampling 校准 S 分布
- **结果**: External ρ 从 0.7446 崩塌至 0.570（Δρ = -0.175）
- **结论**: 问题不是"严重度分布不一致"，而是语义标尺的可迁移性与视觉机制一致性不足

**发现 3：评测单元必须从"图片"提升到"簇/独立场景"**
- 图片层面 Bootstrap: CI 宽度 0.0094 (1.31%)
- 簇层面 Bootstrap: CI 宽度 0.0923 (12.40%)，**放大 9.8x**
- **结论**: Cluster Bootstrap 是跨域测试更可信的做法（方法论贡献）

### 1.2 经验贡献（领域启示）

> **核心论点**（严谨版）:
> 生成式合成对**精细严重度回归任务**的主要风险，在于其引入的"视觉退化机制—标签语义"不一致以及**合成域特有的捷径特征**，导致跨域排序能力下降。然而，在**形态由真实 mask 支撑、且训练目标以时序一致性/稳定性为主**（而非绝对标尺回归）时，合成纹理仍可作为有效的弱监督信号。
>
> **对粗粒度任务的推论**:
> 对更粗粒度的任务（如脏污二分类），合成扩增的成功概率**可能更高**，但仍需通过独立跨域评测排除合成捷径带来的泛化风险。**不能直接推出"可靠"的结论**。
>
> **启示**:
> 生成数据的合理用途应当是：
> 1. **一致性/稳定性约束**（时序训练）
> 2. **自监督学习**（预训练）
> 3. **而非直接替代**真实监督标尺（在精细回归任务中）

### 1.3 战略转型逻辑

```
旧方向: SD 作为静态增强 → 直接提升 MAE/ρ → 失败（标尺污染、排序退化）
      ↓
关键洞察: 时序训练的目标是"帧间一致性"，而非"绝对标尺准确性"
      ↓
新方向: SD 用于时序训练 → 提升时序稳定性 → 符合物理特性（脏污固定、背景变化）
```

## 二、SD 在时序任务中的合理性分析

### 2.1 为什么 WoodScape 不直接用于时序训练

| 限制 | 具体情况 | 影响 |
|------|---------|------|
| 序列长度不足 | 每种脏污最多 3-4 张 | 无法形成稳定的短期序列 |
| 背景变化有限 | 大部分来自同一场景的连续帧 | 背景多样性不足 |
| 时序标注缺失 | 没有明确的"脏污固定"标签 | 时序监督信号弱 |

**结论**: 时序稳定性学习需要 T 足够大（至少 8-16 帧，最好 30+ 帧）来稳定估计"抖动/漂移"，WoodScape 无法满足。

### 2.2 SD 在时序任务中更合理的物理依据

**时序模块的关键监督信号**：
- **不是**每帧的绝对严重度
- **而是**跨帧一致性

**物理先验**：
1. **脏污（镜头前景薄层）**: 在短期近似静止 → Ŝ_t, Ĝ_t 应平稳
2. **背景（场景）**: 随时间变化 → 模型不应把背景变化误认为脏污变化

**关键优势**：这类目标只依赖**相对关系**，对"绝对标尺是否偏一点"不敏感，因此 SD 的主要缺陷（标尺污染）可以规避。

**边界条件（关键）**：

1. **SD 不参与绝对标尺监督**
   - 时序损失 L_temp 只作用于跨帧一致性
   - Global head 的强监督回归只用真实 WoodScape
   - 防止 SD 重新污染标尺

2. **纹理"拟真"要有可量化的 QC**
   - Spill（mask 外污染率）: < 5%
   - 纹理/对比度统计范围: [0.1, 0.8]
   - 人工审查 HQ 300 作为高质量子集
   - 论文中需提供示例对比面板支撑"拟真"主张

### 2.3 关键闭环：SD 不参与"标尺学习"，只参与"稳定性约束"

**硬规则**：
- ✅ **监督标尺**（global head 的绝对回归）只用真实 WoodScape
- ✅ **SD 只用于**产生序列并施加时序一致性损失（自监督/弱监督）
- ❌ **SD 不参与**任何会影响绝对标尺的训练

## 三、数据构造方案

### 3.1 素材准备

#### 3.1.1 脏污层库

| 子集 | 数量 | 来源 | 用途 |
|------|-----:|------|------|
| **HQ** | 300 | 人工筛选的高质量样本 | 主实验（高可信序列） |
| **LQ** | ~8660 | 其余 SD8960 样本 | Ablation（质量影响对比） |

**HQ 筛选标准**（已有人工审查）：
- 纹理逼真度: 与真实脏污高度相似
- 边缘自然: 无明显的合成痕迹
- 透明度合理: 符合物理光学特性

#### 3.1.2 背景序列库

**来源选项**：
1. **自有干净视频帧**（优先）
   - 来自真实车载视频
   - 覆盖多种场景（城市、高速、不同天气）

2. **WoodScape 不同场景的 clean 帧**
   - 如果存在序列数据
   - 需确保场景多样性

### 3.2 序列生成原则（论文方法章节核心）

**核心假设**: 脏污相对镜头固定，场景随车辆移动变化

#### 3.2.1 数学形式

对每条序列 i，固定一个脏污层 (α, D)，其中：
- **α**: 脏污透明度（整个序列不变）
- **D**: 脏污纹理（整个序列不变）

背景 I_t 随时间变化，合成公式：

```
I'_t = (1 - α) ⊙ I_t + α ⊙ Blend(I_t, D)
```

其中 Blend 函数确保脏污与背景的光学融合。

#### 3.2.2 质量门槛（简单但硬性）

不再用"baseline 误差"筛选（避免自蒸馏），改用更贴近"序列一致性"的 QC：

| QC 指标 | 阈值 | 物理含义 |
|---------|------|---------|
| 背景污染率 spill | < 5% | mask 外改动很小 |
| 脏污对比度 | [0.1, 0.8] | 防止过黑或过于遮挡 |
| 纹理自然度 | 人工审查 | HQ 子集已筛选 |

#### 3.2.3 序列参数

| 参数 | 值 | 说明 |
|------|-----:|------|
| 序列长度 T | 16 或 32 | 足够显著体现背景变化 |
| 初始序列数量 | 500-2000 条 | 取决于背景视频帧的长度与多样性 |
| 帧率 | 10-30 fps | 与真实车载视频一致 |

### 3.3 数据集划分

```
SD_seq_HQ: 基于 HQ 300 构造的序列（主实验集）
SD_seq_LQ: 基于 LQ 8660 构造的序列（Ablation 对比集）
```

## 四、训练策略

### 4.1 模型架构

基于现有 Dual-Head 架构，新增时序模块：

```
输入序列 [I_{t-T+1}, ..., I_{t-1}, I_t]
        ↓
Shared Backbone (ResNet18, 从 baseline 加载)
        ↓
特征序列 [F_{t-T+1}, ..., F_{t-1}, F_t]
        ↓
┌─────────────────────────────────────┐
│   时序编码模块（Phase 3 新增）        │
│  ┌────────────┐    ┌────────────┐   │
│  │ Temporal   │ →  │  Temporal  │   │
│  │ Conv(k=3)  │    │  Attn (可选) │  │
│  └────────────┘    └────────────┘   │
└─────────────────────────────────────┘
        ↓
时序感知特征 F_temporal
        ↓
    ┌─────┴─────┐
    ↓           ↓
Tile Head   Global Head
```

### 4.2 损失函数设计

#### 4.2.1 总损失（关键：SD 不影响绝对标尺）

```
L_total = L_static + μ · L_temporal

其中:
L_static = L_tile^{real} + λ · L_glob^{real}    # 只来自 WoodScape
L_temporal = L_tempS^{sd} + η · L_tempG^{sd}     # 只来自 SD 序列
```

**关键设计**：
- ✅ WoodScape 数据: 计算完整损失 L_tile + L_glob
- ❌ SD 序列数据: **只计算时序损失**，不计算 L_glob
- μ: 时序损失权重（建议 0.1）
- η: tile vs global 时序损失的平衡权重

#### 4.2.2 时序损失定义

**1. 全局稳定性损失**:

```
L_tempS = (1 / (B·(T-1))) · Σ_{b,t} |Ŝ_{b,t+1} - Ŝ_{b,t}|
```

**物理含义**: 脏污固定时，全局严重度预测应平稳

**2. Tile 稳定性损失**:

```
L_tempG = (1 / (B·(T-1))) · Σ_{b,t} ||Ĝ_{b,t+1} - Ĝ_{b,t}||₁
```

**物理含义**: 脏污固定时，tile 级覆盖率预测应平稳

### 4.3 三阶段训练策略

#### Stage 1: 基线预训练（已完成）

| 参数 | 值 |
|------|-----|
| 数据 | WoodScape Train_Real (3200) |
| 损失 | L_tile + λ·L_glob + μ_cons·L_cons |
| 输出 | 预训练 backbone + dual head |
| 性能 | Test_Ext ρ = 0.7446 (簇层面) |

#### Stage 2: 时序模块训练（Phase 3A，新增）

| 参数 | 值 |
|------|-----|
| 数据 | SD_seq_HQ（基于 HQ 300） |
| 损失 | **仅** L_temp = L_tempS + η·L_tempG |
| Backbone | **冻结**（防止 SD 影响特征空间） |
| 时序模块 | **训练**（Temporal Conv 或 Attention） |
| Global Head | **冻结**（关键：防止 SD 影响标尺） |
| 预期 | 时序稳定性提升，静态性能持平 |

**冻结策略（保险方案）**：
- 冻结 backbone 全部参数
- 冻结 global head 全部参数
- 只训练时序模块的参数
- 完全隔离 SD 对标尺的影响

#### Stage 3: 端到端微调（Phase 3B，可选）

| 参数 | 值 |
|------|-----|
| 数据 | 混合: WoodScape + SD_seq |
| 损失 | L_static + μ·L_temporal |
| Backbone | 解冻但用小 lr |
| 时序模块 | 继续训练 |
| Global Head | 解冻但用小 lr |
| 关键 | WoodScape 的 L_glob 会"拉回"标尺 |

**保险设置**：
- μ（时序权重）: 0.1（小权重，避免覆盖静态监督）
- lr_static: 1e-4（正常学习率）
- lr_temporal: 3e-4（稍大，保证时序学习）

## 五、评测体系

### 5.1 新增时序稳定性指标（核心贡献）

#### 5.1.1 全局抖动指标

```
J_S = (1 / (T-1)) · Σ_{t=1}^{T-1} |Ŝ_{t+1} - Ŝ_t|
```

**物理含义**: 同一脏污条件下的预测波动程度

**工程意义**: 抖动越小 → 误报越少 → 用户体验越好

#### 5.1.2 Tile 抖动指标

```
J_G = (1 / (T-1)) · Σ_{t=1}^{T-1} ||Ĝ_{t+1} - Ĝ_t||₁
```

**物理含义**: tile 级预测的帧间一致性

#### 5.1.3 报警抖动率

```
F_Alarm = (#{t: Ŝ_t 跨越阈值又回退}) / T
```

**工程含义**: 频繁的"报警→解除→再报警"是典型的不良体验

### 5.2 静态性能指标（安全带）

虽然主目标转为稳定性，但仍必须报告静态性能**不显著恶化**：

| 测试集 | 指标 | 基线 | Phase 3 目标 |
|--------|------|------:|-------------:|
| WoodScape Test_Real | Tile MAE | 0.0396 | < 1.2x 基线 |
| WoodScape Test_Real | Global MAE | 0.0187 | < 1.2x 基线 |
| **External Test_Ext** | **ρ (簇层面)** | **0.7446** | **不显著下降** |

**关键**: Phase 3 的成功标准不是提升 MAE/ρ，而是**不显著损害**的同时提升稳定性。

### 5.3 Ablation 实验设计

#### Ablation 1: 时序模块结构

| 实验 | 时序模块 | L_temp | 预期作用 |
|------|---------|--------|---------|
| Baseline | ✗ | ✗ | 参考基准 |
| + Temporal Conv | ✓ (Conv) | ✗ | 验证时序编码 |
| + Temporal Conv + L_temp | ✓ (Conv) | ✓ | 验证时序损失 |
| + Temporal Attn + L_temp | ✓ (Attn) | ✓ | 验证 Attention |

**目的**: 证明"合成质量"对稳定性学习的影响，支撑 HQ 筛选的必要性。

|------|------------|----------|---------------|
| Baseline | 训练 | 训练 | - |

def generate_sequence(dirt_layer, background_frames, T=16, alpha=None):
    """
    Args:
        dirt_layer: (mask, texture) from HQ 300
        background_frames: list of T background images
        T: sequence length
        alpha: transparency (if None, use original)
    Returns:
        sequence: list of T synthesized frames
        labels: list of (G_t, S_t) for each frame

    sequence = []
    for bg_t in background_frames:
        I_prime_t = synthesize(dirt_layer, bg_t, alpha)
        sequence.append(I_prime_t)
    return sequence

```python
class TemporalConv1D(nn.Module):
    def __init__(self, in_channels=512, kernel_size=3, dropout=0.1):
        super().__init__()
        self.conv = nn.Conv1d(in_channels, in_channels,
                             kernel_size, padding=kernel_size//2)
        self.dropout = nn.Dropout(dropout)

    def forward(self, x):
        # x: [B, T, C, H, W]
        B, T, C, H, W = x.shape
        x = x.permute(0, 3, 4, 1, 2)  # [B, H, W, T, C]
        x = x.reshape(B*H*W, C, T)    # [B*H*W, C, T]
        x = self.conv(x)              # [B*H*W, C, T]
        x = x.dropout(x)
        x = F.relu(x)
        x = x.reshape(B, H, W, T, C)  # [B, H, W, T, C]
        x = x.permute(0, 3, 4, 1, 2)  # [B, T, C, H, W]
        return x.mean(dim=1)          # [B, C, H, W] 时间平均
```

```python
def evaluate_temporal_stability(model, sequence_dataloader):
    """计算 J_S, J_G, F_Alarm"""
    j_s_list = []
    j_g_list = []
    f_alarm_list = []

    for sequence_batch in sequence_dataloader:
        # sequence_batch: [B, T, C, H, W]
        outputs = model(sequence_batch)  # [B, T, ...]

        # 计算抖动
        for b in range(outputs.shape[0]):
            S_hat = outputs[b, :, 'S']  # [T]
            G_hat = outputs[b, :, 'G']  # [T, 4, 8, 8]

            J_S = (S_hat[1:] - S_hat[:-1]).abs().mean()
            J_G = (G_hat[1:] - G_hat[:-1]).abs().mean()

            j_s_list.append(J_S)
            j_g_list.append(J_G)

    return {
        'J_S_mean': np.mean(j_s_list),
        'J_G_mean': np.mean(j_g_list),
    }
```

## 七、论文写作要点

### 7.1 核心贡献表述（严谨版）

**贡献 1：提出并验证 cluster-level 评测方法**
- 发现图片层面 Bootstrap 低估 CI 10-15x（违反独立性假设）
- 簇层面方法更稳定可信，正确处理序列内相关性
- **方法论贡献**：为有重复样本的大测试集提供了统计框架

**贡献 2：系统揭示合成数据在严重度标尺任务中的多重失效机制**

不只是"严重度标签不匹配"，而是：

| 失效机制 | 量化证据 |
|---------|---------|
| 视觉机制—标签语义不一致 | D-only 校准后仍崩塌（Δρ = -0.175） |
| 合成域捷径特征 | 标尺压缩比 1.17-1.34x |
| 成像链统计不一致 | 所有 SD 模型 Test_Ext ρ 显著下降 |
| 标尺语义漂移 | 跨域排序退化（Δρ = -0.097 ~ -0.171） |

**关键发现**：边际分布对齐**不足以**修复这些问题，说明问题不是"严重度分布不一致"，而是更深层的**语义标尺可迁移性与视觉机制一致性**不足。

**经验贡献**：为合成数据在退化类任务中的使用提供了边界条件与失效模式分析。

**贡献 3：提出利用生成数据的时序稳定性学习策略**

核心论点（审稿友好版）：

> 我们观察到，基于 SD 的合成样本在**精细严重度回归任务**中容易引入"视觉退化机制—标签语义"不一致以及**合成域特有的捷径特征**，从而导致跨域排序能力下降；然而，在**形态由真实 mask 支撑、且训练目标以时序一致性/稳定性为主**（而非绝对标尺回归）时，合成纹理仍可作为有效的弱监督信号。对于更粗粒度的任务（如脏污二分类），合成扩增**潜在更可行**，但仍需通过独立跨域评测排除合成捷径带来的泛化风险。

方法创新点：
- SD 用于"一致性约束"而非"绝对标尺"
- 物理先验（脏污固定、背景变化）
- 冻结策略防止标尺污染
- 可量化的 QC 标准（spill、对比度、人工审查）

## 九、边界条件与风险提示（重要）

### 9.1 SD 用于时序训练的边界条件

**必须满足的条件**：

| 条件 | 具体要求 | 违反后的风险 |
|------|---------|-------------|
| **不参与绝对标尺监督** | SD 只计算时序损失 L_temp，不计算 L_glob | 重新污染标尺，复现 Phase 2 失败 |
| **形态由真实 mask 约束** | 使用 WoodScape 的真实 mask 作为形状 | 形态不真实，时序学习无效 |
| **纹理有可量化 QC** | Spill < 5%，对比度 [0.1, 0.8]，人工审查 | 质量无法控制，实验不可复现 |

**关键**：只有同时满足这些条件，SD 用于时序训练才是合理且安全的。

### 9.2 对"粗粒度任务更可行"的限定

**原观点**：
> 对更粗粒度的任务（如脏污二分类），SD 作为数据扩增的成功概率更高。

**限定条件**（必须写入论文）：
1. **仍需跨域验证**：不能直接推出"可靠"，必须通过独立外域测试
2. **捷径风险仍然存在**：即使二分类，SD 特有的纹理风格也可能被模型学习为捷径
3. **成像统计一致性**：如果 SD 合成与真实外域的相机/ISP 差异大，二分类也可能失败

**严谨表述**：
> 对粗粒度任务，合成扩增的**潜在风险可能降低**（因为标签语义更宽松），但**不能直接推导为"可靠"**。仍需通过严格的跨域评测来验证是否存在合成域捷径。

### 9.3 审稿人可能的质疑点与回应

| 潜在质疑 | 回应要点 |
|---------|---------|
| "SD 不就是质量不够好吗？" | 不只是质量问题，还有**视觉机制—标签语义不一致**和**合成域捷径特征**。D-only 校准后仍崩塌证明了这一点。 |
| "时序训练就能解决 SD 的问题吗？" | 时序训练**不解决**SD 的标尺问题，而是**规避**它。时序目标（一致性）不依赖绝对标尺。 |
| "你们的 HQ 300 够吗？" | HQ 300 用于主实验，LQ 8660 用于 Ablation。质量差异的影响是实验的一部分，不是隐藏的变量。 |
| "为什么不做端到端的 SD 标注训练？" | Phase 2 已证明：任何形式的 SD 混入监督都会损害跨域排序。时序训练的关键是**隔离** SD 对标尺的影响。 |
| "你们的时序稳定性指标有工程意义吗？" | J_S 和 J_G 直接对应"误报抖动"和"报警反复"，是车载系统中的关键用户体验问题。 |

### 9.4 实验设计的自我批判

**我们承认的局限**：

1. **HQ 300 是人工筛选的**
   - 承认：主观性难以完全避免
   - 回应：提供可量化的 QC 指标（spill、对比度）+ 示例对比图

2. **时序训练没有完全排除 SD 影响**
   - 承认：即使冻结 global head，backbone 仍可能在 SD 上更新
   - 回应：Phase 3A 采用完全冻结策略作为对照

3. **真实视频评测数据可能有限**
   - 承认：车载视频数据获取成本高
   - 回应：SD 序列上的评测仍是有效验证（物理一致性是内生的）

4. **二分类推论没有直接实验**
   - 承认：这是基于理论分析的推论
   - 回应：明确标注为"未来工作"或"推论"，不做过度宣称
